# OrbitForge Market Edge

OrbitForge should not try to win by copying every feature from every AI tool.

It should win by solving the larger problem around high-trust agentic engineering work.

## What similar tools already do well

### Continue

- rules, prompts, slash commands, and reusable assistants
- broad customization around model choice and prompts

Reference points:

- https://docs.continue.dev/customize/overview/
- https://docs.continue.dev/customize/rules
- https://docs.continue.dev/actions/context-selection

### Roo Code

- custom modes
- MCP-heavy tooling
- checkpoints, task tracking, and powerful editor-native agent flows

Reference point:

- https://docs.roocode.com/

### n8n agents

- agent nodes inside a broader workflow engine
- strong automation and integration story

Reference points:

- https://docs.n8n.io/integrations/builtin/cluster-nodes/root-nodes/n8n-nodes-langchain.agent/
- https://docs.n8n.io/advanced-ai/chat-hub/

## What OrbitForge should own

### 1. Lifecycle stewardship

Most tools help users run an agent.

OrbitForge should help users steward the entire lifecycle:

- intake
- context packing
- disagreement
- approval
- validation
- release
- publish

### 2. Cross-surface parity

The same lifecycle contract should travel across:

- hosted builder
- CLI
- VS Code
- desktop

### 3. Local-first legitimacy

Ollama, LM Studio, and OpenAI-compatible local endpoints should feel first-class, not like fallback modes.

### 4. Trust before fluency

The output should push toward:

- mission board
- implementation order
- validation
- remaining risk

### 5. Portable ecosystem artifacts

OrbitForge blueprints should be:

- plain JSON
- versionable
- reviewable
- forkable
- runnable in multiple surfaces

## Viral loops OrbitForge can create

### Blueprint loop

Users create a blueprint, run it locally, improve it, and share it.

### Review loop

Teams use OrbitForge for risky reviews and circulate the proof-heavy output internally.

### Local-first loop

Developers who cannot or do not want to rely on hosted providers share OrbitForge because it respects local inference.

### Contributor loop

Open-source contributors extend starter kits and lifecycle components instead of just filing ideas.

## Near-term moves with the highest leverage

- publish the VS Code extension in Marketplace and Open VSX
- add a public blueprint pack registry format
- add mission replay / checkpoints across surfaces
- add evidence bundle export for PRs and release reviews
